Installation

Get up and running within minutes with Composer.

If you don't know about Composer, I would highly recommend visiting their site and learning how to use it in your projects.


Composer Projects

First, copy this line into your project's composer.json file, in the "require": {} section

"khill/lavacharts" : "{{ site.composer_version }}"

Next, run composer to download and install

{% highlight shell startinline %} $ composer update {% endhighlight %}

Finally, require('vendor/autoload.php'); in your project and you are good to go!

Laravel

Service Provider

First step, register the service provider by adding the class namespace to the providers array in app/config/app.php

{% highlight php %} "providers" => [ ... 'Khill\Lavacharts\Laravel\LavachartsServiceProvider', ] {% endhighlight %}

If you are using PHP 5.5+, then you can use the new ::class syntax to reference the Service Provider.

{% highlight php %} Khill\Lavacharts\Laravel\LavachartsServiceProvider::class {% endhighlight %}

Facade / Alias

The functionality is based upon the single instance of the library, so it is strongly recommended to add the Lava alias for ease of use throughout your application.

{% highlight php startinline %} "aliases" => [ ... 'Lava' => 'Khill\Lavacharts\Laravel\LavachartsFacade', ] {% endhighlight %}

Same as the service provider, if you are using PHP 5.5+, you can use the ::class syntax.

{% highlight php %} Khill\Lavacharts\Laravel\LavachartsFacade::class {% endhighlight %}

Remember, many examples show $lava-> but while using Laravel, enable the alias and use \Lava:: instead.
