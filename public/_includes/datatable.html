Taking a closer look

A column can be defined as an array with the following properties

array( string $type [, string $colDesc [, string $colLabel ]] )

There are many ways to add columns to the DataTable, all following the above syntax

addColumn( string $type [, string $colDesc [, string $colLabel ]]) returns DataTable
addColumn( array( string $type [, string $colDesc [, string $colLabel ]]) ) returns DataTable
addColumns( array( /* of column arrays, see above */ ) ) returns DataTable

Also provided are simplified, alias methods, which will automatically fill the type and colLabel.

addDateColumn( string $colDesc ) returns DataTable
addNumberColumn( string $colDesc ) returns DataTable
addStringColumn( string $colDesc ) returns DataTable

  • First, create a new DataTable object using the method from Lavcharts.
  • Next, add some columns, defining the structure of the chart's data.
    • 'date' type column accepts a Carbon Object, or any string that Carbon::parse() can interpret.
    • 'number' type column accepts any integer or float value.
  • The Carbon library is included with Lavacharts, so just add use Carbon\Carbon; to the file where you are creating the DataTable, to create Carbon objects for date columns.
    • Note, you could also just pass a parsable datetime string, instead of including the Carbon class.
  • In this example, the first column is the horizontal axis, then then next two columns are the two sets of data.
  • The order of arguments are as follows for the addColumn method [data type, label (opt), id (opt), formatter (opt)]
  • The addRow() method signature, follows the order in which the columns were added.
    So here, data[0] is for 'day', data[1] is for 'projected' and data[2] is for 'official'
